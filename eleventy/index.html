Breve reseña de Eleventy

Mkdocs no esta orientado a la creacion de un blog, sino de documentacion en linea. En un blog, los articulos se organizan conforme a sus metadatos, ya sea por su orden cronologico de entrada o mediante palabras claves (tags) relacionados con su contenido, y se presentan en paginas html como listas de enlaces. Como Mkdocs no soporta estas caracteristicas, hemos cambiado a Eleventy.

Eleventy es un generador de sitios web basado en javascript. Utiliza distintos lenguajes de plantilla (template languages), para construir las paginas html. Los dos lenguajes basicos son liquid y nunjucks.

Liquid se basa en el lenguaje yaml situado en la cabecera del documento (front matter), y permite aplicar sus datos dentro de plantillas incluidas en el html. Nunjucks es similar, pero adicionalment admite el uso de javascript en el front matter.

---
title: Birds
birds:
  - robin
  - finch
  - sparrow
---
<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>{{ title }}</title>
</head>
<body>
    <h1>{{ title }}</h1>

    <ul>
        {% for bird in birds %}
        <li>{{ bird }}</li>
        {% endfor %}
    </ul>

</body>
</html>

El resultado de este bloque al construir la pagina html seria.

<!DOCTYPE html>
<html>
<head>
<meta charset="utf-8">
<title>Birds</title>
</head>
<body>
<h1>Eleventy Title from Data</h1>
<ul>

    <li>robin</li>

    <li>finch</li>

    <li>sparrow</li>

</ul>
</body>
</html>

Sobre el concepto que soporta este ejemplo basico se construye Eleventy. Las plantillas se usan para dividir el html de la pagina final en distintos componentes, desde un armazon base en el archivo base.njk, hasta pequeños fragmentos de numjucks que vehiculan los metadatos html para crear elementos de una lista, y se engloban en una estructura for.

Una vez creada esta estructura, la informacion incorporada en nuestros archivos markdown fluye atraves de estas plantillas incorporando caracteristicas y generando distintos tipos de paginas html.

Estructura de proyecto con Eleventy

Por su complejidad, utilizar correctamente Eleventy lleva tiempo. Un camino que nos permite utilizar el programa inmediatamente es instalar el modelo basico, que la propia Eleventy pone a nuestra disposicion.

En su estructura, existen distintos componentes direrenciados:

  • La plantilla base y sus asociadas directas,en el directorio _include, escritas en nunjucks con cabeceras yaml.

  • Las entradas del blog, en el directorio posts, escritas en markdown con cabeceras yaml.

  • Puntos de acceso del navegador, en el directorio base. Estas paginas presentan listas cronologicas y son mostradas desde los enlaces navegador de la aplicacion. Utilizan el tag post y generan una lista con los articulos que incluyen dicho tag. Los datos del tag se encuentran almacenados en una clase nativa del programa llamada collections.

  • Listados de tags, en el directorio base. Similar a al anterior, su objetivo es mostrar el listado de todos lo tags. En este caso de ha incluido una nueva coleccion con el listado de tags en el archivo de javascript, .eleventy.js. Posteriormentese ha utilizado como en el caso anterior.

La clase collections se vuelve un elemento fundamental al manejar los tags, y en la formacion de estruturas for que generen listas en nuestro html

<!DOCTYPE html>
<html>
<head>
    <title>Blog Posts</title>
</head>
<body>
    <ul>
        {% for post in collections.post -%}
        <li>{{ post.data.title }}, {{ post.data.date }}</li>
        {% endfor -%}
    </ul>
</body>
</html>

Esta breve orientacion muestra los caminos que sigue la informacion desde los archivo de liquid, atraves de plantillas nunjucks y clases collection, hasta la creacion de las paginas html dentro de este modelo basico. Con el uso de este modelo, la documentacion oficial, este video-tutorial y esta otra documentacion que incide en aspectos mas teoricos, podremos manejar el programa.

Hay que tener en cuenta que el modelo basico tiene todas funcionalidades propias de un blog, y que modificar el aspecto estetico, corre en paralelo al programa y se basa en css ligado a nuestra plantilla base.

Paginación

Antes de terminar, me gustaria añadir a los conceptos clave de las plantillas y las colecciones, el concepto de paginación.

La paginacion se incluye en el front matter de la siguiente manera,

---
title: Fish Paged
pagination:
    data: fishList
    size: 2
---
{% include _head.html %}
<h1>{{ title }}</h1>
<ul>
    {% for fish in pagination.items -%}
    <li>{{ fish }}</li>
    {% endfor -%}
</ul>
{% include _foot.html %}

Esto permite, igual que hacia el ejemplo del inicio, introducir una estructura de datos dentro de las plantillas de html, solo que en lugar de usar elementos procedentes del front matter, lo introduciriamos desde un archivo json situado en un directorio predefinido llamado _data. En el ejemplo seria un archivo llamado fishList.json y referenciado desde data en el front matter.

El mismo metodo tambien se puede usar para acceder a las colecciones de programa. Este sitema es usado en el archivo page-list.njk de nuestro modelo basico de Eleventy.

Integrar en otro sitio web

No seria extraño que nos interesase integra el blog para complementar otra web, y la forma de hacerlo es modificado parcialmente la ruta de acceso antes de construir el blog. Esto se realiza en el archivo .eleventy.js. modificando el componente pathPrefix.

Conclusion

Es evidente que crear un blog no es tan sencillo tecnicamente como crear una pagina de documentacion, ya que hay que crear pagina html mucho mas flexibles. Utilizando el modelo básico de Eleventy, tendremos un buen punto de partida para publicar el blog inmediatamente al tiempo que vamos aprendiendo como adaptarlo a nuestras necesidades.
