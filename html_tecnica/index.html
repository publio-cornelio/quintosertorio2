Creacion de paginas web

La creación de paginas webs se basa en dos lenguajes de marcado con etiquetas, html y css, y un lenguaje de programación, javascript (js)

Un lenguaje de marcado es aquel que junto al texto incorpora etiquetas que nos indican que función tiene dicho texto en dicho documento, por ejemplo, en html <p></p> nos indica que el texto contenido es un párrafo.

Css es un lenguaje de marcado que utiliza las etiquetas creadas en html, para modificar los atributos estéticos del documento y mostrarnos el texto en pantalla con una distribución, tamaño y estéticas determinadas.

Por contra, js es un lenguaje de programación, con estructuras de control, variables, funciones; su objetivo es facilitar en las paginas web, una estética dinámico y un comportamiento interactivo con el usuario.

Lenguajes de marcado. HTML y CSS

Html es la columna vertebral de cualquier documento en la web. Presenta numerosas etiquetas nativas. El núcleo básico son términos propios de un procesador de textos como encabezado, párrafo, sección; que se amplían con metadatos, con secciones propias de una pagina web como el navegador, y con etiquetas orientadas a incrustar imágenes, contenido multimedia, tablas o formularios.

La presentación por defecto de un archivo html sigue un patrón flow layout. Lo cual significa, que hay elementos block que se sitúan uno de bajo de otro y elementos inline (el texto) que se se sitúa uno a continuación del otro.

Pero hablar de elementos en bloque o en linea, es pasar a la terminología del css. Esta habla de la distribución en el espacio de los distintos elementos, de elementos contenedores y elementos contenidos, de los cambios en la disposición en función del tamaño de la pantalla.

Css utiliza las etiquetas propias del html para seleccionar los elementos y distribuirlos. Y a su vez influye en la creación de elementos auxiliares que actuando como contenedores faciliten dicha distribución. Con este fin, etiquetas como división o span, que no definen una posición precisa dentro de la estructura de un texto, cumplen una función esencial detrás de escena.

Llegados a este punto parece útil poner un ejemplo sencillo con los elementos esenciales que componen la relación entre html y css.

Html

 <div class="division"></div>
        <h1>Esto es un encabezado</h1>
        <p class="texto_azul">Lorem ipsum dolor sit amet consectetur. Fugit, id.</p>
        <p class="texto_azul">Lorem ipsum dolor sit, amet consectetur adipisicing elit.</p>
        <p>Lorem ipsum dolor sit amet.</p>
 </div>


Css

div {}
.division{}
h1 {}
p {}
.texto_azul{
    color: blue;
}

En este ejemplo es interesante resaltar la relacion entre el continente div y el contenido organizado en un cabecero y tres parrafos. Css puede hacer referencia tanto a las etiquetas nativas de html, como a las clases precedidas por un punto. En este caso hemos agrupado dos parrafos y les hemos dado un atributo diferenciador.

El sistema es sencillo y muy flexible, y exige asignar los atributos de manera ordenada, partiendo de aquellos generales que afectan a todo el documento para continuar hacia los mas especificos.

La clave en el uso del Css esta en los atributos que determinan la disposicion espacial. Existen varios modelos de disposicion espacial, cada uno con sus atributos. Conocer el modelo es imprescindible para entender la terminologia y aplicacion de sus atributos. Pero esto forma parte del proximo articulo.

Javascript y librerias de componentes.

Js es el componente del trio mas dificil de manejar, si bien se pueden hacer buenas paginas web solo con los otros dos.

En la web de presentacion el uso de js es minimo. Activa la escucha de tres eventos: el movimiento del scroll, el click de un boton y el cambio de tamaño de pantalla. En funcion de la entrada de datos controla la trasparencia del cabezero. Para facilitar la referencia a las etiquetas y clases del html, el archivo html incorpora una libreria de componentes jquery.

Estas librerias de componentes son archivos css o js creados por terceros, que incorporan distintas utilidades como facilitar el uso de javascript o mejorar la calidad de los elementos graficos.

Conclusion

Esta introduccion busca ser un marco de referencia antes de comenzar a crear nuestra primera pagina web. La escritura basica de html y css es sencilla, y la complejidad esta hacer crecer un proyecto de forma ordenada sin entrar en contradicciones entre los distintos elementos.
