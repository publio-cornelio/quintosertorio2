Integración del proyecto.

En este punto, disponemos de dos componentes diferenciados, una pagina web responsive creada directamente con html y css, y un sitio web con una estructura de directorios y paginas webs.
Ambos componentes los vamos a unir en un mismo directorio, vamos a incorporar los enlaces para que interactuen y por ultimo, los vamos a poner en linea en un dominio propio.

Directorios y enlaces.

La estructura de directorios se forma al introducir el directorio del soporte tecnico dentro del de la pagina web en la forma:

pagina-web
    |____ soporte-tecnico

En mi caso, tengo un git en el directorio de salida habitual del generador de sitios web. Por ello, genero un directorio de salida especifico con la orden: mkdocs build -d soporte-tecnico

Antes de generar dicha salida, modifico el archivo yaml con una entrada en el navegador del tipo -Volver a pagina-web: '../', que permite ir al directorio raiz de pagina-web. Como no transcribe bien esta referencia en el archivo index.html de soporte-tecnico, a posteriori hay que reescribirla en dicho archivo.

En el la propia pagina web, tambien escribo la refencia de soporte tecnico del navegador con el directorio incorporado /soporte-tecnico/

Y con estos cambios quedan conectadas, se puede comprobar en el servidor local y empujar a la red.

Creación de dominio propio

Hoy en dia existe un proveedor de dominios web gratuitos llamado freenom, que nos puede servir para ir conociendo la tecnica de configuracion de un dominio propio.

El paso mas sencillo es conectarse a su sitio web, comprobar si un dominio esta disponible. Esto se hace en un buscador que la pagina pone a tu disposicion.

Si bien encontraras dominios disponibles, no podras seleccionarlos en esta busqueda inicial. Para seleccionarlos deberas volver a buscar, pero esta vez incluyendo la extension seleccionada en el termino de busqueda.

A partir de aqui inicias un procedimiento de registro en freenom, y el siguiente obstaculo surge al configurar netlify para utilizar el dominio.

Primero, tienes que indicar el nombre del dominio personalizado a netlify en el apartado de custom domains. Tras incluir el nombre, netlify te ofrece un desplegable de opciones donde aparece un panel de DNS (servicios de nombre de dominio), aqui puedes encontrar los nombres de servidores de netlify. Ahora hay que copiar estos nombres y pegarlos en freenon en el apartado de gestion de tu dominio. En este momento tu dominio queda dirigido hacia netlify, este realiza las pertinentes comprobaciones del certificado de seguridad y tu nombre de dominio queda activo.

Conclusión

A partir de este momento todo el soporte tecnico de nuestro sitio web esta realizado. Resta incorporar un segundo generador de sitios web, que funciona como el primero salvo que utilizaremos un tema diferente.
